用mockjs模拟数据

发表于 2017-09-26 | 分类于 js |

TL;DR

  • 安装 npm install mockjs
  • 数据模板定义规范
    • 属性名和生成规则必须用引号包围, 用|隔开
    • 属性值类型:number,string,boolean,object,array,function,regexp
    • 生成规则主要是 'name|min-max': value和'name|count': value, 属性值类型决定其含义
    • number: min-max之间的随机数   无后者不过加了浮点类型
    • string: min-max之间的随机重复次数   重复次数count
    • boolean: value出现的概率是 min/(min+max)    false和true均为1/2
    • object: 随机抽取min-max个属性   抽取count个属性
    • array: min-max之间的随机重复次数    1的话表示随机抽取一个值,其他值表示重复count次
    • function: 就是执行函数,this是属性所在的对象
    • regexp: 用正则生成字符串
  • 数据模板定义规范:用 @ 来标识其后的字符串是占位符 且占位符优先使用数据模板 中的属性
  • Mock.mock( rurl?, rtype?, template|function( options ) ) 生成数据
  • Mock.random() 占位符 随机生成想要的数据,比如email

h5和app交互的一些坑

发表于 2017-09-14 | 分类于 js |

安卓不兼容js最新的方法

这个特别注意，一不小心就犯了这个错。果子一般都支持。所以如果果子可以，安卓不可以，优先考虑这个问题

window对象延迟的问题

安卓一般情况下，会注入一个对象，这个在页面一开始加载就有了，比如安卓在window上注册了某个方法，基本可以直接使用。
不过偶尔安卓工程师可能会手动改这个注入顺序，这个需要沟通。

但是苹果，肯定是延迟注入的，是等页面加载完之后的不定时注入对象，自测了下不太准，50ms左右吧。
比如苹果提供某个方法，不能一开始就直接使用，需要延迟使用，一般监测到这个方法存在了，才能使用。

参数不对的问题

有时候h5如果传给app的参数不对的话，会出现各种问题。所以一定再三确认，参数没问题。
说起来，安卓不接受对象形式，所以一般用JSON。stringify转换对象传过去，但是果子可以。

脑洞各种意外的坑

如果别的安卓机都没问题，部分安卓机有问题，请发挥脑洞。
好像遇到过，魅族部分机型对于长字符串发送不了请求。。。

贝塞尔曲线的理解

发表于 2017-07-10 | 分类于 js |

TL;DR

Bezier曲线的由来

1962年，法国工程师贝塞尔发表，他运用贝塞尔曲线来为汽车的主体进行设计

Bezier曲线的作用

Bezier曲线是用一系列点控制曲线状态的。主要分为

  • 数据点：确定曲线的起始和结束位置
  • 控制点：确定曲线的弯曲程度

举例理解：想在AC（起始点和结束点）之间画一个曲线，用B点（控制点）控制这个曲线的弯曲程度

但是控制点是可以多个的，比如两个控制点。以此类推，可以有很多个。

起点和终点都只有一个，但是控制点可以多个，甚至是0，0的时候就是直线啦！

Bezier曲线的原理

为什么几个点就可以得到一个曲线？
先说一个控制点的情况，如图所示：

  1. A/B/C三点是确定的
  2. 在AB上任取一点D，得到ratio = AD/AB
  3. 再由BE/BC = ratio 得到E点
  4. 连接DE，同理DF/DE = ratio得到F
  5. 而F点就是曲线上的一点，当然凭着这一点是无法得到整条曲线的
  6. 于是，再来一遍，重新取D点得到新的F点，以此类推，如图

    道理是一样的，在AB上任取一点E，得到曲线上的J点。
    AE/AB = BF/BC = CG/CD = EH/EF = FI/FG = HJ/HI

理解Bezier曲线的公式

一次贝塞尔曲线


一次贝塞尔曲线（也是线性贝塞尔曲线）公式：B(t) = (1 - t) * P0 + tP1

  • t表示在 P0P1/P0P1之间任取一点P2，t = P0P2/P0P1，也就是比例，公式里的P0和P1同步表示其x轴坐标或者y轴坐标。
  • 已知P0的坐标是（a,b），P1的坐标是（c,d），那么假设P2的坐标是（x,y）
  • (1-t)/(c-x) = t/(x-a) => x = (1-t)c + ta
  • 同理 y = (1-t)d + tb
  • 于是简写成 B(t) = (1 - t) * P0 + tP1

二次贝塞尔曲线


二次贝塞尔曲线（也是线性贝塞尔曲线）公式：

  • t同上
  • 在P0P1上的点是A，在P1P2上的点是B，在AB上的点是C，C也就是曲线上的一点。
  • A ： tP1 + (1-t)P0
  • B ： tP2 + (1-t)P1
  • C ： tB + (1-t)A 将A、B换成上式，进行合并同类项
