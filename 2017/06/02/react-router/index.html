以下是2.x的版本，router的中文是路由器，route的中文是路由，也就是router控制route
首先 index.html

重点index.js

  • node_moudles/react-router/lib/index.js 查看可用的api
  • Router组件本身只是一个容器，真正的路由要通过Route组件定义。
  • 必须有Router(有history属性)，Route(有path和component属性)。
  • 组件，如果是纯函数的话，可以直接用箭头函数
  • path的路径不分大小写，也就是 “/hello” 等价于 “/Hello”
  • 子路由也可以不写在Router组件里面，单独传入Router组件的routes属性。

Link

react中，也有类似的a标签就是Link。其to属性就相当于href属性。
当location.pathname包含to的值时，Link上面的activeClassName或activeStyle就能生效。
还有个IndexLink，一般和activeClassName或activeStyle相配合，只有当该路由被渲染的时候，才会active。
如果不使用IndexLink，也可以加上onlyActiveOnIndex={true}这个属性，达到相同的效果。

嵌套路由

嵌套路由，就是访问 /dear/huahua。

IndexRoute表示父路径自动加上该组件
上面一串的route表示
/ => 且将这个组件里面的{props.children} 换成
/dear => 且将这个组件里面的{props.children} 换成
/dear/hua => 且将这个组件里面的{props.children} 换成
里面的{props.children} 换成

  • 若当前的Router标签的path路径cur，其父标签是parent，那么访问parent/cur路径的时候，相当于渲染父标签的component，且将其{props.children} 换成当前标签的component。
  • IndexRoute相当于访问parent的时候，自动将parent组件里面的{props.children}自动换成IndexRoute的组件了
  • 除了根目录带着/，其他路径不带/
  • 可以看下打印出来的props里面有啥

params

就是路径里面有变量，路径不一样，显示的内容不一样。本质是通过props.params获得路径里面的变量值。

  • “/(:message)”记得括号
  • {props.params.message || ‘ee’}容错的判断
  • Link标签和变量路径的结合使用，如列表页面跳到详情页面

query

query和params有点类似。

  • 组件都可以获得到query和params的值
  • 都可以和Link配合使用
  • {(obj && obj.attr) || ‘none’} 表示obj存在的就是obj.attr，obj不存在的话就’none’
  • 属性是对象的时候

Redirect

标题为params的demo稍微改下：

  • Redirect访问from的时候就跳到to
  • 注意顺序，这里将Route放在Redirect的前面的时候，/w是可以被渲染出来的。
  • Redirect和嵌套有关，规则等同Route

components

Route上components=obj的属性，该父标签的props里面就会有components的键，props.key对应着obj.key的组件。
目前这个属性，应该必须和组件嵌套配合使用。

setRouteLeaveHook

切换路由之前要做啥。

  • this.props.router.setRouteLeaveHook(this.props.route,this.callback)表示离开this.props.route这个路由的时候，做this.callback
  • callback默认有一个参数nextLocation，值是obj
  • callback返回的字符串是询问句，确定就会返回true跳转，取消返回false不跳转，所以直接返回false的时候就无法跳转。
