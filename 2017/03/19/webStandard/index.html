web标准

Posted on 2017-03-19 | In 前端规范 |

前言

对于IT行业，设备的多样性与信息的多样性需要标准来保证它们之间的沟通与协作。我们可能在手机、打印机、数码相机之间交互数据。如果没有统一的标准，那么现在所有的信息都是孤立的、不可分享、不可重用的；我们需要浪费大量的人力物力重新为新设备建立数据；我们需要为每一次的多系统应用做专门的开发。

这个统一的标准就是—XML。XML简单说就是一种“有规定格式的文档”，只要我们的设备或系统产生的数据都遵守这种格式，就可以实现互相的数据交换、分享与协作。

自从1998年国际万维网组织(W3C.org)推出XML 1.0规范以来，已经有大量的XML标准在我们的生活中应用，例如我们用手机订阅的天气预报、股票行情，这些数据都是从相应的系统获得然后通过XML格式转换发送给你的。

那么，网页(web)是否也应该遵循XML标准呢？答案是肯定的。

我们有99%的网站是采用HTML制作的，而HTML并不符合XML格式。因此这些网页信息都很难适应未来新设备和数据共享的要求。怎么办呢？国际万维网组织(W3C.org)在HTML基础上，按照XML格式制定了新的规范XHTML1.0，只要通过简单的改变，就能将HTML转为XHTML，从而实现向XML的过渡。同时，为了使你的页面信息更加容易被搜索和重用、XHTML的代码需要结构更加更清晰、标签更加有语义，W3C推荐使用CSS来控制表现，以实现内容与表现的相分离。

什么是web标准？

为了实现大量HTML信息向XML标准的过渡，W3C和ECMA制定的一系列的技术规范，目前主要包括XHTML1.0、CSS2.0、DOM1.0和ECMA JavaScrit。web标准不仅仅是一个规范，而是一系列规范的总称。

W3C是什么？

W3C是“World Wide Web Consortium”的缩写，中文称万维网组织。是一个专注于“领导和发展web技术”的国际工业行业协会。成立与1994年。W3C已经有超过500家的会员–包括微软、美国在线(Netscape的母公司)、苹果电脑、Adobe、Macromedia、SUN 以及各类主流硬件、软件制造商和电信公司。

W3C主要工作是研究和制定开放的规范(事实上的标准)，以便提高web相关产品的互用性。W3C的推荐规范的制定都是由来自于会员和特别邀请的专家组成的工作组完成。工作组的草案(Drafts)在通过多数相关公司和组织同意后提交给W3C理事会讨论，正式批准后才成为“推荐规范(Recommendations)”发布。更多的信息您可以访问W3C的网站：www.w3.org

针对网页的web标准

网页主要由三部分组成：结构（Structure）、表现（Presentation）和行为（Behavior）。对应的标准也分三方面：结构化标准语言主要包括XHTML和XML，表现标准语言主要包括CSS，行为标准主要包括对象模型（如W3C DOM）、ECMAScript等。

推行的原因

  1. 浏览器和访问网页的设备都很多，为了 web 更好地发展，在开发新的应用程序时，浏览器开发商和站点开发商需要共同遵守web标准。
  2. Web 标准也可以使站点开发更快捷，更令人愉快。开发人员不必为了得到相同的结果，而挣扎于多版本的开发。
  3. 标准的 web 文档更易被搜索引擎访问，更易被转换为其他格式，更易被程序代码访问（比如 JavaScript 和 DOM）。

总结

web标准是为了将大量HTML信息向XML标准过渡制定的一系列规范。从而方便各设备能准确且统一读取这些信息。
