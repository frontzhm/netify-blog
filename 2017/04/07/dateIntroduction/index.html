Date对象的理解和使用

发表于 2017-04-07 | 分类于 js |

创建Date实例用来处理日期和时间。
Date对象是基于1970年1月1日（世界标准时间）起的毫秒数。

TL;DR

创建date实例

创建实例有以下四种形式：没有参数第一个，至少两个参数是第四种情况，参数没有默认0。新建的时间都是本地时间，不是世界标准时间即格林威治时间。我们国家时间比格林威治快8小时。

  • new Date();
  • new Date(value);
    value代表自1970年1月1日00:00:00 (世界标准时间) 起经过的毫秒数。
  • new Date(dateString);
    表示日期的字符串值。该字符串应该能被 Date.parse() 方法识别（符合 IETF-compliant RFC 2822 timestamps 或 version of ISO8601）。
  • new Date(year, month[, day[, hour[, minutes[, seconds[, milliseconds]]]]]);
    year代表年份的整数值。为了避免2000年问题最好指定4位数的年份; 使用 1998, 而不要用 98.
    注意：month代表月份的整数值从0（1月）到11（12月）
    day代表一个月中的第几天的整数值，从1开始。
    hour代表一天中的小时数的整数值 (24小时制)。后面的字面上意思。

备注

月份：”Jan” / “Feb” / “Mar” / “Apr” / “May” / “Jun” / “Jul” / “Aug” / “Sep” / “Oct” / “Nov” / “Dec”
周：”Mon” / “Tue” / “Wed” / “Thu” / “Fri” / “Sat” / “Sun”

静态方法

Date.now()

返回自1970年1月1日 00:00:00 UTC到当前时间的毫秒数，类型为Number。
相当于new Date().getTime()

Date.parse(dateString)

解析一个表示某个日期的字符串，并返回从1970-1-1 00:00:00 UTC 到该日期对象（该日期对象的UTC时间）的毫秒数,其最后三位始终是000。
如果参数不能解析为一个有效的日期，则返回NaN。类型为Number。
其实就是隐式调用new Date(dateString)。

Date.UTC(year,month[,date[,hrs[,min[,sec[,ms]]]]])

接受以逗号隔开的日期参数，返回1970-1-1 00:00:00 UTC到指定的日期之间的毫秒数。
仔细理解下，日期参数是本地时间 localTime，其转换为格林威志时间 glTime，将1970-1-1 00:00:00 UTC设为 startGl，其转化为本地时间是 startLocal
那么 这个返回的结果是 res。时间差始终是一样的。
res = localTime - startGl = localTime - ( startLocal - 时间差) = (glTime + 时间差) - startGl
