Vue的介绍

Posted on 2017-04-01 | In js |

Vue.js 是一套构建用户界面的渐进式框架。
Vue 采用自底向上增量开发的设计。
Vue 的核心库只关注视图层，它不仅易于上手，还便于与第三方库或既有项目整合。
另一方面，当与单文件组件和 Vue 生态系统支持的库结合使用时，Vue 也完全能够为复杂的单页应用程序提供驱动。

TL;DR

声明式渲染： 和 v-bind:title=”title”
条件：v-if=”isSeen”
循环：v-for=”item in items”
事件绑定：v-on:click=”someFn”
表单元素双向绑定：v-model=”value”
组件：Vue.component(‘componentName’,{props:[item],templates:”< li></ li>”})

声明式渲染

两种声明：

  • 采用简洁的模板语法来声明式的将数据渲染进 DOM：,数据和 DOM 被绑定在一起，一切都是响应式的
  • 绑定 DOM 元素属性 v-bind:title = "cnt",v-bind将这个元素节点的 title 属性和 Vue 实例的 cnt 属性保持一致”。

条件与循环

v-if="seen" 控制切换一个元素的显示。
vue不仅可以绑定 DOM 文本到数据，也可以绑定 DOM 结构到数据。而且，Vue 也提供一个强大的过渡效果系统，可以在 Vue 插入/更新/删除元素时自动应用过渡效果。
v-for = 'todo in todos' 指令可以绑定数组的数据来渲染一个项目列表

处理用户输入

v-on:click="reverseMessage" 指令绑定一个调用 Vue 实例方法的事件监听器
reverseMessage没有触碰 DOM - 所有的 DOM 操作都由 Vue 来处理，你编写的代码只需要关注底层逻辑。
v-model="value" 指令，使表单输入和应用状态间的双向绑定变得轻而易举。双向意味着，当html那边改变时，vue实例里面的相应值也会变化

组件化应用构建

在 Vue 里，一个组件本质上是一个拥有预定义选项的一个 Vue 实例，在 Vue 中注册组件很简单，接着将应用分割成了两个更小的单元，子单元通过 props 接口实现了与父单元很好的解耦。我们现在可以进一步为我们的 todo-item 组件实现更复杂的模板和逻辑的改进，而不会影响到父单元。

与自定义元素的关系

  1. Vue 组件支持ie9+,必要时，可以被包装在原生自定义元素之内
  2. Vue 组件提供了纯自定义元素所不具备的一些重要功能，最突出的是跨组件数据流，自定义事件通信以及构建工具集成。
