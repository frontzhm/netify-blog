vue的表单控件绑定

发表于 2017-05-17 | 分类于 js |

TL;DR

  • v-model在表单控件元素上创建双向数据绑定。v-model="message"双向的意思是，input这边的输入值变化会让message变化，而message变化会让input的值变化。
  • v-model在文本框内就是文本框的值
  • v-model在复选框里，默认是布尔值，当有多个复选框时，要设置为数组，选中某项则数组中就会出现此项的值
  • v-model在选择列表里，放在select标签上，等于选中的option的值
  • v-model一般是input事件，修饰符lazy，可以将其变成change事件
  • 修饰符三种：lazy、number、trim（基本都是文本框）

请直接前往官网阅读

基础用法

你可以用 v-model 指令在表单控件元素上创建双向数据绑定。它会根据控件类型自动选取正确的方法来更新元素。尽管有些神奇，但 v-model 本质上不过是语法糖，它负责监听用户的输入事件以更新数据，并特别处理一些极端的例子。
注意：v-model 并不关心表单控件初始化所生成的值。因为它会选择 Vue 实例数据来作为具体的值。
注意：对于要求 IME （如中文、 日语、 韩语等） 的语言，你会发现那v-model不会在 ime 构成中得到更新。如果你也想实现更新，请使用 input事件。

input(type="text",v-model="message"),双向的意思是，input这边的输入值变化会让message变化，而message变化会让input的值变化。
还有个地方，一般我们写表单元素会有name值，但是这边v-model绑定的名字其实相当于name值，所以这边的表单元素没有name值

修饰符

onchange事件在内容改变（两次内容有可能还是相等的）且失去焦点时触发；
oninput当 元素的value属性值由输入设备改变时，就会触发input事件。

  • .lazy 在默认情况下， v-model 在 input 事件中同步输入框的值与数据 (除了 上述 IME 部分)，但你可以添加一个修饰符 lazy ，从而转变为在 change 事件中同步
  • .number 如果想自动将用户的输入值转为 Number 类型（如果原值的转换结果为 NaN 则返回原值），可以添加一个修饰符 number 给 v-model 来处理输入值：这通常很有用，因为在 type=”number” 时 HTML 中输入的值也总是会返回字符串类型。
  • .trim 如果要自动过滤用户输入的首尾空格，可以添加 trim 修饰符到 v-model 上过滤输入

v-model与组件

HTML 内建的 input 类型有时不能满足你的需求。还好，Vue 的组件系统允许你创建一个具有自定义行为可复用的 input 类型，这些 input 类型甚至可以和 v-model 一起使用！要了解更多，请参阅自定义 input 类型
