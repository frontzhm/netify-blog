TL;DR

  • 变量类型分原始类型和引用类型。判断方法是typeof和instanceof。区别是按值传递和按共享传递。
  • 理解原型。每个引用对象都可拓展属性，都有pro属性，属性值指向其构造函数的prototype。每个函数都有prototype属性，属性值是一个普通对象。读取一个对象的属性值的时候，找不到就会去其proto里找，顶层是Object.prototype.__proto__ = null。某构造函数F创建的对象obj，obj.proto.proto= Fn.prototype.proto=Object.prototype.

变量类型

原始类型：string number boolean null undefined symbol(es6新加的)。

object不属于原始类型。

判断原始类型的方法：typeof。能得到的结果有string number boolean undefined symbol function object。

注意点：typeof null的结果是object，这是个bug，null是原始值，非引用类型。还有function是可以检测到的。其他引用类型都是object。

判断引用类型的方法：instanceof。用于实例和构造函数的对应。比如[1,2] instanceof Array。

原始类型和引用类型的区别：按值传递和按共享传递。

测试变量类型

1因为Number类型的a是按值传递的，而Object类型的b是按共享传递的。a不变b变化了。（1 {value:’new’}）。
2虽然obj本身是个引用类型的变量（对象），但是内部的a和b一个是值类型一个是引用类型，a的赋值不会改变obj.a，但是b的操作却会反映到obj对象上。({a:1,b:[1,2,3,4]},2,[1,2,3,4])

JS 中这种设计的原因是：按值传递的类型，复制一份存入栈内存，这类类型一般不占用太多内存，而且按值传递保证了其访问速度。按共享传递的类型，是复制其引用，而不是整个复制其值（C 语言中的指针），保证过大的对象等不会因为不停复制内容而造成内存的浪费。

原型和原型链

理解原型

  • 所有引用类型都具有对象特性，即可自由扩展属性（注意null并不是引用类型，自然也就不行）。

  • 所有函数都有prototype属性，属性值也是一个普通对象。

  • 所有引用类型都有proto属性，属性值是一个普通对象，指向其构造函数的prototype属性值。

  • 当试图得到一个对象的某个属性时，如果这个对象本身没有这个属性，那么会去它的proto（即它的构造函数的prototype）中寻找。判断是不是自己的属性hasOwnProperty。

原型测试题

f.__proto__即Foo.prototype，没有找到toString，继续往上找

f.__proto__.__proto__即Foo.prototype.__proto__。Foo.prototype就是一个普通的对象，因此Foo.prototype.__proto__，其构造函数是Object，Object.prototype，在这里可以找到toString

因此f.toString最终对应到了Object.prototype.toString

这样一直往上找，你会发现是一个链式的结构，所以叫做“原型链”。如果一直找到最上层都没有找到，那么就宣告失败，返回undefined。最上层是什么 —— Object.prototype.proto === null

原型链中的this：所有从原型或更高级原型中得到、执行的方法，其中的this在执行时，就指向了当前这个触发事件执行的对象。因此printName和alertName中的this都是f。

作用域和闭包

执行上下文

全局执行上下文：js脚本先解析后执行。解析的时候先创建一个 全局执行上下文 环境，变量先暂时赋值为undefined，函数则先声明好可使用。然后才是执行。通俗的说，执行代码的时候，function和var的名词已经存在了，即便顺序在后面。

函数执行上下文：一个函数在执行之前，也会创建一个 函数执行上下文 环境，跟 全局上下文 差不多，不过 函数执行上下文 中会多出this arguments和函数的参数。参数和arguments好理解，这里的this咱们需要专门讲解。

this的值是在执行的时候才能确认，定义的时候不能确认！ 为什么呢 —— 因为this是执行上下文环境的一部分，而执行上下文需要在代码执行之前确定，而不是定义的时候。

this执行会有不同，主要集中在这几个场景中

  • 作为构造函数执行，构造函数中
  • 作为对象属性执行，上述代码中a.fn()
  • 作为普通函数执行，上述代码中fn1()
  • 用于call apply bind，上述代码中a.fn.call({name: ‘B’})

作用域

es6之前，JS 没有块级作用域，只有全局作用域和函数作用域。这就是为何 jQuery、Zepto 等库的源码，所有的代码都会放在(function(){….})()中。因为放在里面的所有变量，都不会被外泄和暴露，不会污染到外面，不会对其他的库或者 JS 脚本造成影响。这是函数作用域的一个体现。ES6 中开始加入了块级作用域，使用let定义变量即可。所以es6之后可以不用var。

作用域链

自由变量：当前作用域没有定义的变量。自由变量如何得到 —— 向父级作用域寻找。如果父级也没呢？再一层一层向上寻找，直到找到全局作用域还是没找到，就宣布放弃。这种一层一层的关系，就是 作用域链 。

闭包

自由变量将从作用域链中去寻找，但是 依据的是函数定义时的作用域链，而不是函数执行时。
闭包主要有两个应用场景：

  • 函数作为返回值
  • 函数作为参数传递

异步

这俩到底有何区别？—— 第一个示例中间的步骤根本没有阻塞接下来程序的运行，而第二个示例却阻塞了后面程序的运行。前面这种表现就叫做 异步（后面这个叫做 同步 ），即不会阻塞后面程序的运行。

JS 需要异步的根本原因是 JS 是单线程运行的，即在同一时间只能做一件事，不能“一心二用”。

一个 Ajax 请求由于网络比较慢，请求需要 5 秒钟。如果是同步，这 5 秒钟页面就卡死在这里啥也干不了了。异步的话，就好很多了，5 秒等待就等待了，其他事情不耽误做，至于那 5 秒钟等待是网速太慢，不是因为 JS 的原因。

看个demo

这是一个很有迷惑性的题目，不少人认为100ms之后，由于a变成了false，所以while就中止了，实际不是这样，因为JS是单线程的，所以进入while循环之后，没有「时间」（线程）去跑定时器了，所以这个代码跑起来是个死循环！

前端异步的场景，一旦遇到代码会先执行其他内容，最后执行异步:

  • 定时 setTimeout setInterval
  • 网络请求，如 Ajax <img>加载

ES6 箭头函数中的this和普通函数中的this区别

箭头函数里的this跟父作用域的 this一致。

es6模块化的理解

  • 如果只是输出一个唯一的对象，使用export default即可。
  • 如果想要输出许多个对象，就不能用default了，且import时候要加{…}。

class和构造函数的区别

其实都是类啦。更加易懂的写法而已。这里注意属性私有，方法共有。也就是方法写在prototype上。class写法的时候constructor即构造器，初始化实例时默认执行。

实现继承的时候特别能体现class的好。

  • 继承注意3c原则，call constructor create。mdn的解释
  • ES5 的继承，实质是先创造子类的实例对象this，然后再将父类的方法添加到this上面（Parent.call(this)）。
  • ES6 的继承机制完全不同，实质是先将父类实例对象的属性和方法，加到this上面（所以必须先调用super方法），然后再用子类的构造函数修改this。
  • super关键字的使用，简单的说就是constuctor里就是父类构造函数，在方法里可以做对象使用，相当于父类的prototype。

set 和 map

  • Set 类似于数组，但数组可以允许元素重复，Set 不允许元素重复，重复的会被忽略。用来数组简单的去重有没有。
  • Map 类似于对象，但普通对象的 key 必须是字符串或者数字，而 Map 的 key 可以是任何数据类型
