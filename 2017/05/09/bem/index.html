使用BEM

发表于 2017-05-09 | 分类于 css |

TL;DR

  1. bem表示block、element、modifier。
  2. 块是独立的，可以随意挪动和嵌套。每个块有自己的html、css、js以及图片和文档
  3. 元素是block的组成部分，不能脱离block的上下文。
  4. 修饰符作为一个块或元素的一种属性，代表这个块或元素在外观或行为上的改变。
  5. 用BEM树描述页面。
  6. 常用的bem命名.blockName__elementName-modifierName { }
  7. 用sass写bem命名.blockName {&__elementName{&-modifierName{...}} }
  8. 原子类可以和块配合使用，但不轻易用在元素上
  9. 状态一般可以用通用类名如is-active，.blockName.is-active{display:block}

BEM是什么

BEM代表块（Block），元素（Element），修饰符（Modifier）。BEM和OOP是相似的。其是一种用代码和一系列模式来描述现实情况的方法，它只考虑程序实体而无所谓用什么语言
我们使用BEM的原则创建了一个前端开发技巧和工具的集合，这样我们就能快速构建一个网站，并且保证他们长久的可维护性。
一个块是一个独立的实体，就像应用的一块“积木”。任意放在任何的位置，也能嵌套在其他块里。
一个元素是他们应该属于块的一部分，具有某种功能。元素是依赖上下文的：它们只有处于块的上下文中时才有意义。

描述页面和模板的意义

块和元素构成页面的内容。它们不仅仅是被呈现在一个页面上，它们的排列顺序也同样重要.
块（元素）彼此之间可能遵循着某种顺序。

每个块和元素都应该有一个可以识别的关键字，也就是名字

一个项目中块名是唯一的。块范围内的一种元素的名字也是唯一的。但可以多次出现。

为独立的css类命名

块必须在css里名字也是唯一的。
块在css的类名字就是自己的关键字。
元素在css里面的类名是 块名+符号+元素名

eg:
blockName-elementName
block-name-element-name

独立块模板

对于模板引擎来说，块的独立性意味着，块名对应唯一的html
当模板引擎在某个模板中遇到一个块的时候可以准确地把这个块转换成HTML。因此每个块都应该有自己的模板。

块修饰符

经常需要创建一个和已存在的块非常相似的块，只是外观或行为有些改变。这样引入修饰符（modifier）的概念。
修饰符作为一个块或元素的一种属性，代表这个块或元素在外观或行为上的改变。
一个修饰符有一个名字和一个值。多个修饰符可以同时使用。

BEM的命名

一个块有其html css js甚至图片和文档。
BEM的命名规矩大概就是分清b/e/m,我们只需保留BEM的思想，其命名规范可以任意变通。

对于命名过长的问题，服务器端有gzip且编辑器一般会提示。

原子类和BEM

在组件开发中其实不推荐使用原子类。
原子类更适合应用在实际页面中，这是因为页面变动大而且不可复用。

header可以封装成一个模块，但它复用程度不高，不能算是组件，所以即使使用原子类也没有关系。在项目中，使用原子类之前应该考虑一下，这个场景是否变动大而且不可复用，如果是的话，我们可以放心的使用原子类。

组件应该是“自洽的”，其本身就应该构成了一个“生态圈”，也就是说，他几乎不需要外部供给，自给自足就能够运转下去。
