vue深入了解组件篇

  • prop特性
  • 非prop特性，$attr接受
  • 自定义事件，$listeners
  • slot
  • keep-alive

组件注册

  • 组件在js那边是驼峰式，在html结构里是羊肉串式，因为html不区分大小写
  • 全局注册Vue.component('component-a', { /* ... */ })，注册之后可以用在任何新创建的 Vue 根实例 (new Vue) 的模板中，然后html结构里可以写组件名，但是实际中一般一个应用只有一个vue实例，所以尽量避开
  • 局部注册，new Vue({ components: { 'component-a': ComponentA} })，每个vue实例中传入components
  • 模块系统里，用import导入模块。基础组件，配合webpack的require.context使用

prop

  • prop通常数组形式，props: ['title', 'likes']
  • 可以传递静态或者动态prop，动态用:xx="i"，默认是字符串类型，当时布尔类型或者数组或者对象，都需要用v-bind
  • 传入对象所有属性的时候，<blog-post v-bind="post"></blog-post>，v-bind:id="post.id" v-bind:title="post.title"
  • 单向数据流，如果子组件需要数据但只是本地修改的话，最好定义一个本地的 data或者computed 属性并将这个 prop 用作其初始值。props: ['initialCounter'], data () { return { counter: this.initialCounter } }，！！！对象或者数组的话注意会改变原始数据
  • prop验证，形式如下代码，！！！注意prop 会在一个组件实例创建之前进行验证，所以实例的属性 (如 data、computed 等) 在 default 或 validator 函数中是不可用的。
  • prop验证的type可以是构造函数，当然原始的有String Number Boolean Array Object Date Function Symbol
  • 非prop特性的数据，也就是不用prop接受的数据，对于绝大多数特性来说，从外部提供给组件的值会替换掉组件内部设置好的值。所以如果传入 type="text" 就会替换掉 type="date" 并把它破坏！庆幸的是，class 和 style 特性会稍微智能一些，即两边的值会被合并起来，从而得到最终的值。
  • 如果不希望组件的根元素继承特性，可以Vue.component('my-component', { inheritAttrs: false })，注意class和style不受控制
  • $attrs可以在组件内任意元素拿到一个组件的特性名和特性值，<input v-bind="$attrs" >

自定义事件

  • 自定义事件名请始终用羊肉串形式
  • 一个组件上的 v-model 默认会利用名为 value 的 prop 和名为 input 的事件，但可以通过model来避免这样的冲突Vue.component('base-checkbox', { model: { prop: 'checked', event: 'change' }, props: { checked: Boolean }, template: `<input type="checkbox" v-bind:checked="checked" v-on:change="$emit('change', $event.target.checked)" > ` })
  • 将原始事件绑定到组件，Vue 提供了一个 $listeners 属性，它是一个对象，里面包含了作用在这个组件上的所有监听器，v-on="$listeners"，当然需要补充的话，可以在computed里Object.assign({},this.$listeners,其他监听器)，避免用.native，以防止根元素变化
  • 修饰符.sync，子组件需要修改父组件的时候，通常父组件定义事件，子组件$emit触发，现在可以用update，也可以进一步用sync修饰符<text-document v-bind:title.sync="doc.title"></text-document>等价于<text-document v-bind:title="doc.title" v-on:update:title="doc.title = $event" ></text-document>
    组件内部this.$emit('update:title', newTitle)

slot

  • slot有自己的名字，用到父组件的属性之类的

动态组件&异步组件

  • keep-alive可以缓存组件的状态<keep-alive> <component v-bind:is="currentTabComponent"></component> </keep-alive>
  • 异步组件一般和webpack合用

处理边界情况

  • $root，$parent，$child，$refs，$provide
  • $on $off $once
  • 循环引用，创建文件树
  • $forceUpdate 强制更新
  • v-once使用的时候注意场景，以及注释，尽量避免大量使用
